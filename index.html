Висновки

Предметним середовищем для цього проекту є розробка інтернет-магазину "Cedre", який слугуватиме зручною платформою для покупців меблів. Основною метою проекту є надання користувачам можливості переглядати, фільтрувати та купувати різні види меблів, такі як дивани, ліжка, шафи тощо. Застосунок орієнтований на широке коло користувачів, що шукають сучасні та функціональні меблі для свого дому або офісу. Основний акцент зроблено на зручність користувацького інтерфейсу та забезпечення максимальної інформативності для покупців.

Головна сторінка: Коли користувач заходить на головну сторінку онлайн-магазину меблів, його зустрічає основна інформація про магазин. На цій сторінці розміщено промо-банери, які відображають акції, новинки та популярні товари. Головна сторінка слугує відправною точкою для користувача, де він може дізнатися про актуальні пропозиції та перейти до різних розділів магазину.

Фільтри товарів: Однією з ключових функцій застосунку є система фільтрів, яка дозволяє користувачам швидко знаходити потрібні меблі за різними критеріями.

Фільтри включають:
1. Фільтрація за типом меблів: Користувач може обрати тип меблів з переліку (диван, крісло, шафа, ліжко). Вибір типу меблів автоматично завантажує відповідні матеріали та кольори, доступні для цього типу.

2. Фільтрація за матеріалом: Після вибору типу меблів користувач бачить список доступних матеріалів. Користувач може обрати один або кілька матеріалів, за якими хоче фільтрувати товари.

3. Фільтрація за кольором: Відображаються доступні кольори для вибраного типу меблів. Користувач може обрати один або кілька кольорів.

4. Фільтрація за ціною: Відображаються різні цінові діапазони (до $50, $50-$100, $100-$200, $200-$300, понад $300). Користувач може обрати ціновий діапазон, що його цікавить.

Фільтри дозволяють користувачам швидко звузити вибір і знайти меблі, які відповідають їхнім вимогам і бюджету.

Каталог: Каталог товарів відображає список меблів, що відповідає обраним фільтрам. Користувач може сортувати товари за різними критеріями, такими як ціна, матеріал або колір. Кожен товар у каталозі містить зображення, назву і ціну.

Кошик: Кошик відображає всі товари, які користувач додав для покупки.

Тут користувач може:
1. Переглядати список обраних товарів з їхніми параметрами.
2. Редагувати кількість товарів або видаляти товари з кошика.
3. Переглядати загальну вартість замовлення.

Front-end: React: Основна бібліотека для побудови користувацького інтерфейсу. React Router: Для маршрутизації між сторінками.
CSS: Стилізація інтерфейсу.
HTML: Структура сторінок.

Back-end: Firebase: Використовується для зберігання даних про товари та користувачів.
Firestore: Для роботи з базою даних в реальному часі.


Зовнішні бібліотеки та інструменти: Firebase Authentication: Для автентифікації користувачів.

Стек технологій Для реалізації фронт-енд частини проекту "Онлайн-магазин меблів" було обрано використання React, React Router, CSS/SCSS та HTML. Кожна з цих технологій відіграє важливу роль у створенні ефективного та зручного інтерфейсу для користувачів.

React: React є основною бібліотекою для побудови користувацького інтерфейсу в цьому проекті. Вибір React зумовлений його численними перевагами, що включають:

1. Компонентний підхід: React дозволяє розробляти інтерфейс у вигляді незалежних компонентів, які можна багаторазово використовувати та комбінувати. Це спрощує розробку та тестування окремих частин інтерфейсу.

2. Віртуальний DOM: Використання віртуального DOM у React забезпечує високу продуктивність програми, оскільки дозволяє мінімізувати кількість маніпуляцій з реальним DOM. Це значно пришвидшує рендеринг та оновлення інтерфейсу.

3. Односторонній потік даних: React використовує односторонній потік даних, що спрощує розуміння та налагодження програми. Це забезпечує передбачувану поведінку інтерфейсу та спрощує управління станом додатка.

React Router : React Router використовується для маршрутизації між сторінками в додатку. Він забезпечує плавний перехід між різними частинами інтерфейсу без перезавантаження сторінки, що покращує користувацький досвід. Основні переваги React Router включають:

1. Декларативний підхід: React Router дозволяє описувати маршрути у декларативному стилі, що робить код зрозумілішим та легшим для підтримки.

2. Динамічні маршрути: React Router підтримує динамічні маршрути, що дозволяє створювати гнучкі та масштабовані маршрути для різних частин додатка.

3. Легкість інтеграції: React Router легко інтегрується з React, забезпечуючи плавний перехід між компонентами та сторінками.

HTML та CSS: HTML відповідає за створення семантично правильної та зрозумілої структури сторінок, визначаючи розташування та взаємозв'язки між елементами контенту. CSS використовуються для стилізації цих елементів, забезпечуючи їх привабливий зовнішній вигляд та узгодженість з дизайнерськими вимогами.

1. Семантична структура та стильова модульність: HTML дозволяє створювати семантично зрозумілу структуру сторінок, що покращує доступність та SEO-оптимізацію додатка. CSS, у свою чергу, надають можливість створювати модульні та повторно використовувані стилі, що зменшує дублювання коду та спрощує його підтримку.

2. Відокремлення контенту від стилів: Використання HTML для структури та CSS для стилізації забезпечує чітке розмежування контенту від візуального представлення. Це полегшує розробку та підтримку коду, дозволяючи дизайнерам та розробникам працювати над своїми частинами проекту незалежно один від одного.

3. Крос-браузерна сумісність: HTML та CSS підтримуються всіма сучасними веб-браузерами, що гарантує коректне відображення інтерфейсу на різних пристроях та платформах. Це забезпечує користувачам єдиний досвід незалежно від використовуваного браузера.

4. Гнучкість та масштабованість: HTML та CSS забезпечують високу гнучкість у створенні адаптивних дизайнів, що дозволяють інтерфейсу додатка добре виглядати та функціонувати на різних екранах і роздільних здатностях. Це важливо для забезпечення зручності користування на різних пристроях, від мобільних телефонів до настільних комп'ютерів.

Опис структури: Структура бази даних у Firebase забезпечує гнучкість та масштабованість, що є ключовими для ефективного управління інформацією в онлайн-магазині меблів. Колекції та документи дозволяють організувати дані логічно і структуровано, що полегшує доступ та маніпуляцію цими даними у веб-додатку. Нижче наведено детальний опис структури цієї бази даних.

Колекції: Колекції в Firebase є основними контейнерами для документів. В даній базі даних використовуються колекції для різних типів меблів: "Armchair", "Bed", "Sofa", "Wardrobe". Наприклад, колекція "Armchair" містить документи, що описують окремі моделі крісел.

Документи: Документи в колекціях є об'єктами, що зберігають інформацію про конкретні товари. Кожен документ має унікальний ідентифікатор. Наприклад, документ у колекції "Armchair" може містити дані про модель крісла "Harmony".

Структура документів: Кожен документ має поля, що містять інформацію про товар. Поля можуть бути різних типів: рядки, числа, масиви тощо.

Приклад структури документа для товару "Harmony" в колекції "Armchair":
  • color: Колір товару ("Blue");
  • id: Унікальний ідентифікатор товару ("12");
  • img: URL зображення товару ("gs://...");
  • material: Масив, що містить матеріали, з яких виготовлено товар (["Wood", "Textile"]);
  • price: Ціна товару (42);
  • product_name: Назва товару ("Harmony").

ShoppingBag: Колекція "ShoppingBag" містить документи, які представляють товари, додані користувачем до кошика. Кожен документ в колекції "ShoppingBag" (та інших категоріях) має унікальний ідентифікатор, що складається з випадково згенерованого рядка (наприклад, "2da4e331-d0ed-4792-834e-b59ef7d8daea").

Реалізація підключення

Підключення Firebase до React-проекту включає налаштування Firebase, встановлення необхідних бібліотек та реалізацію коду для взаємодії з базою даних. Завдяки цьому, додаток зможе зберігати, отримувати та маніпулювати даними про товари в онлайн-магазині меблів, забезпечуючи ефективну роботу з базою даних у реальному часі.

1. Налаштування Firebase:

Першим кроком для підключення Firebase до нашого React-проекту є створення проекту в консолі Firebase та отримання конфігураційних даних. Ці дані будуть використані для налаштування з'єднання з базою даних.

2. Встановлення необхідних бібліотек:

Після отримання конфігураційних даних ми можемо встановити необхідні бібліотеки, такі як 'firebase' та 'react-firebase-hooks', які допоможуть нам взаємодіяти з Firebase у нашому React-проекті.

3. Реалізація коду для взаємодії з базою даних:

Після налаштування та встановлення необхідних бібліотек ми можемо реалізувати код для взаємодії з базою даних Firebase. Це включає в себе створення, читання, оновлення та видалення даних з бази даних у реальному часі.

Реалізація фільтрування товарів: У проекті реалізовано пошук товарів за обраними параметрами за допомогою функцій фільтрації та сортування. Після того, як користувач обирає тип меблів, матеріал, колір та ціновий діапазон за допомогою відповідних фільтрів на сторінці, дані про обрані параметри передаються в функцію, яка відповідає за відображення відфільтрованих товарів. Ця функція отримує список всіх доступних товарів та застосовує до нього вибрані фільтри, виводячи на екран лише ті товари, які відповідають обраним параметрам. Таким чином, користувач може ефективно знаходити потрібні товари, враховуючи свої вимоги та переваги.

Реалізація кошику

У проекті реалізовано функціонал кошика для товарів, що дозволяє користувачам зберігати обрані товари та керувати ними. Після обрання товару користувач може додати його до кошика за допомогою відповідної кнопки. При цьому обрані товари зберігаються у структурі даних, що відображає кошик. Користувач може переглянути вміст кошика, переглянути кількість доданих товарів та їх загальну вартість. Також реалізовано можливість видалення товарів з кошика та зміни їх кількості. Коли користувач готовий здійснити покупку, він може перейти до оформлення замовлення, де буде надана інформація про товари у кошику та обрані параметри. Така реалізація кошика дозволяє зручно керувати вибраними товарами та легко здійснювати покупки.

Важливим аспектом стало використання різних технологій, таких як React для клієнтської частини, Firebase для бази даних та автентифікації користувачів, інтеграція з API для обробки платежів та роботи з поштою. Крім того, застосунок може зберігати інформацію про попередні покупки користувача, що робить його ще більш зручним і практичним. Він допомагає зекономити час і зусилля користувачів, допомагаючи їм знайти потрібні меблі швидко та ефективно.
